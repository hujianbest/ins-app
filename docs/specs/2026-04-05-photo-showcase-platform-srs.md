# 摄影作品展与约拍网站需求规格说明

- **状态：** 已批准
- **主题：** 摄影作品展与约拍平台
- **评审结论：** 通过（已完成真人确认）
- **关联评审：** `docs/reviews/spec-review-photo-showcase-platform.md`

## 目标

打造一个面向摄影师与模特的作品展播与约拍网站，为注册用户提供大方优雅、全屏视觉冲击感强的个人主页，用于展示作品、建立个人形象并承接约拍机会。

该网站同时服务于两类核心场景：

1. 摄影师或模特向潜在合作对象展示个人作品与风格。
2. 摄影师或模特发布约拍诉求并被其他人发现、联系。

## 范围

首版范围包括：

- 面向摄影师与模特的注册与登录。
- 摄影师个人主页与模特个人主页。
- 公开作品浏览与个人主页浏览。
- 登录用户对作品点赞、收藏个人主页。
- 浏览者通过站内方式联系主页主人。
- 摄影师与模特发布约拍诉求。
- 网站公共首页，以强视觉表现展示精选内容与约拍动态。

## 用户角色

### 摄影师

- 注册并维护个人主页。
- 上传和管理个人作品。
- 接收来自浏览者或潜在合作对象的站内联系。
- 发布约拍诉求。

### 模特

- 注册并维护个人主页。
- 上传和管理个人展示内容。
- 接收来自浏览者或潜在合作对象的站内联系。
- 发布约拍诉求。

### 浏览者

- 未登录时可浏览公开内容。
- 登录后可点赞作品、收藏个人主页、发起站内联系。

## 功能需求

### FR1 账号与身份

- 系统应支持用户注册与登录。
- 系统应支持至少两类可注册身份：摄影师、模特。
- 未登录用户可访问公开首页、公开个人主页、公开作品内容与公开约拍诉求列表。

### FR2 个人主页

- 系统应为摄影师提供个人主页。
- 系统应为模特提供个人主页。
- 个人主页应支持展示主页封面、头像、昵称/名称、个人简介以及公开展示内容。
- 个人主页应具备鲜明视觉表达，优先强调大图、沉浸式浏览和作品氛围。
- 个人主页应支持被其他登录用户收藏。

### FR3 作品与展示内容管理

- 摄影师应能上传、编辑、删除自己的作品内容。
- 模特应能上传、编辑、删除自己的展示内容。
- 每条展示内容至少应支持图片形式展示。
- 展示内容应能在个人主页中公开呈现，并支持单独浏览。

### FR4 公共浏览

- 网站首页应向未登录用户开放浏览。
- 网站首页应展示具有全屏视觉冲击力的核心视觉区域。
- 网站首页应能承载精选摄影作品、精选主页或推荐约拍诉求等公共内容入口。
- 浏览者应能从首页进入摄影师主页、模特主页、作品详情或约拍诉求详情。

### FR5 点赞与收藏

- 登录用户应可对公开展示内容点赞。
- 登录用户应可收藏摄影师主页或模特主页。
- 未登录用户尝试点赞或收藏时，系统应引导其先登录。

### FR6 站内联系

- 登录用户应可通过站内方式联系个人主页主人。
- 联系过程应发生在站内，不要求公开展示微信、手机号等站外联系方式。
- 被联系方应能在站内查看收到的联系请求或消息。

### FR7 约拍诉求发布

- 摄影师可发布约拍诉求。
- 模特可发布约拍诉求。
- 每条约拍诉求至少应包含 `城市` 与 `时间` 两个必填字段。
- 约拍诉求应支持公开展示与浏览。
- 浏览者应能从约拍诉求进入发布者主页，并通过站内方式联系发布者。

## 非功能需求

- 网站整体视觉风格应突出“全屏视觉冲击”与“作品氛围感”，优先保障首页与个人主页的高级感展示效果。
- 公开浏览体验应尽量降低门槛，确保未登录用户可顺畅浏览核心内容。
- 网站应同时适配常见桌面端与移动端浏览器。
- 页面应以图片展示体验为核心，避免界面被低价值信息打断。
- 站内联系信息应默认仅在站内流转，不以公开页面直接暴露站外私人联系方式为前提。

## 约束

- 联系方式限定为站内联系，不以站外即时通讯工具为首版前提。
- 首版必须支持摄影师与模特两类用户都拥有个人展示主页。
- 首版约拍诉求的最小必填信息为 `城市` 与 `时间`。
- 首版重点是展示与联系闭环，不包含交易闭环。

## 范围外内容

以下内容不在首版范围内：

- 在线支付
- 实名认证
- 内容审核后台
- 评论系统
- 作品视频
- 商单撮合

## 验收标准

### AC1 公开浏览

- 未登录用户可以访问首页、摄影师主页、模特主页、公开作品内容与公开约拍诉求。

### AC2 个人主页

- 摄影师注册后可以创建并维护自己的个人主页。
- 模特注册后可以创建并维护自己的个人主页。
- 个人主页包含明显的视觉封面区域，并可承载公开展示内容。

### AC3 展示内容

- 摄影师可以上传至少一条图片作品并在主页公开展示。
- 模特可以上传至少一条图片展示内容并在主页公开展示。

### AC4 点赞与收藏

- 登录用户可以对公开作品点赞。
- 登录用户可以收藏摄影师主页或模特主页。
- 未登录用户在点赞或收藏时会被要求先登录。

### AC5 站内联系

- 登录用户可以从摄影师主页、模特主页或约拍诉求详情页发起站内联系。
- 被联系方可以在站内看到收到的联系内容或联系请求。

### AC6 约拍诉求

- 摄影师可以发布包含 `城市` 与 `时间` 的约拍诉求。
- 模特可以发布包含 `城市` 与 `时间` 的约拍诉求。
- 其他用户可以浏览该诉求并通过站内方式联系发布者。

### AC7 首页体验

- 首页具备明显的全屏首屏视觉区域。
- 用户可以从首页进入作品、个人主页或约拍诉求。

## 开放问题

- 首版是否允许同一账号同时拥有“摄影师 + 模特”双身份，当前规格未强制要求。
- 首版是否需要对点赞、收藏、联系与约拍诉求提供站内通知中心，当前规格未强制要求。
- 首页“精选内容”的排序策略和推荐规则暂未定义，可在设计阶段细化。
