# 网站全站中文化需求规格说明

- 状态: 已批准
- 主题: 网站全站中文化

## 1. 背景与问题陈述

当前摄影作品展与约拍网站已经完成首页、公开浏览链路、认证入口、创作者工作台、互动能力与首页发现增强，但现有站点的用户可见语言仍以英文为主。

这会导致：

- 当前目标用户在进入站点后，需要先理解英文界面，再完成浏览、约拍、登录和联系等操作
- 站点品牌调性、内容场景和交互反馈与中文用户预期不一致
- 已交付的浏览器验证与完成证据无法证明“该网站是中文网站”

本轮需要解决的问题，是在不改变现有核心信息架构和关键路由的前提下，把当前站点面向最终用户的主要可见内容调整为中文，并重新建立可验证的中文浏览基线。

## 2. 目标与成功标准

### 2.1 目标

- 让当前网站的主要用户可见界面默认以中文表达
- 保持现有公开浏览、登录入口、创作者工作台和站内联系流程可继续使用
- 在完成中文化后，用浏览器验证关键用户路径的实际显示结果

### 2.2 成功标准

- 访客访问首页、公开内容页、认证页和工作台时，主要标题、说明、按钮、空态和反馈文案默认为中文
- 页面不再以英文作为主要交互语言，只有品牌名、URL、代码标识或少量行业专有名词可保留英文
- 现有关键用户路径在中文化后仍可正常浏览和操作
- 至少完成一次浏览器级验证，证明中文界面在关键路径中真实可见

## 3. 用户角色与关键场景

### 3.1 主要用户

- 未登录访客：浏览首页、摄影师/模特主页、作品详情和约拍诉求
- 已登录普通浏览者：在公开页进行点赞、收藏、联系并进入收件箱
- 摄影师 / 模特创作者：登录后进入 `studio` 管理公开资料、作品和诉求

### 3.2 关键场景

- 访客打开首页后，应以中文理解品牌引导、发现分区和继续浏览入口
- 访客进入公开主页、作品详情、诉求列表与详情时，应以中文理解内容结构与交互入口
- 用户进入登录 / 注册 / 工作台 / 收件箱时，应以中文理解身份入口、操作按钮、状态提示与空态反馈

## 4. 范围

- 首页与首页发现相关的用户可见文案中文化
- 公开浏览链路中文化，包括：
  - 摄影师主页
  - 模特主页
  - 作品详情页
  - 诉求列表页
  - 诉求详情页
- 认证入口与登录后能力中文化，包括：
  - 登录页
  - 注册页
  - `studio` 落点与子页面
  - `inbox` 收件箱
- 交互反馈中文化，包括：
  - 按钮文案
  - 空态提示
  - 登录拦截提示
  - 收藏 / 点赞 / 联系等反馈文案
- 浏览器验证中文化结果，覆盖至少一个首页入口、一个公开详情入口和一个登录后入口

## 5. 范围外内容

- 多语言切换器
- 完整国际化框架接入或语言包管理后台
- URL 路径本地化或路由结构重命名
- 数据库结构调整
- 重新设计视觉风格或重做信息架构

## 6. 术语与定义

- 全站中文化：当前轮次中，面向最终用户的主要界面表达默认使用中文，而不是英文
- 主要用户可见内容：标题、正文摘要、按钮、标签、导航提示、空态、表单标签、反馈消息、metadata 等用户直接看到的界面文本
- 可保留英文内容：品牌名、URL、技术路径、少量行业通用专有名词或样本数据中不影响主要理解的专名

## 7. 功能需求

### FR-001 首页中文化

#### 需求说明

系统应将首页首屏、品牌说明、发现分区、入口卡片和相关提示文案调整为中文，同时保持现有首页层级和视觉结构不退化。

#### 验收标准

- Given 访客访问首页，When 页面完成渲染，Then 首页主标题、副标题、分区标题和主要按钮应以中文显示。
- Given 访客浏览首页发现分区，When 查看作品、主页、诉求分区，Then 分区标题、说明和空态提示应以中文显示。
- Given 首页已完成中文化，When 用户进入首屏，Then Hero 仍保持当前首页的第一视觉焦点，不因中文化而被结构性破坏。

### FR-002 公开浏览链路中文化

#### 需求说明

系统应将公开浏览链路中的主要界面文案调整为中文，覆盖主页、作品和诉求相关页面，并保持原有公开可访问性。

#### 验收标准

- Given 访客访问摄影师主页或模特主页，When 页面渲染完成，Then 角色标签、栏目标题、联系入口和辅助说明应以中文显示。
- Given 访客访问作品详情页，When 页面渲染完成，Then 作品相关说明、作者信息区和主要交互入口应以中文显示。
- Given 访客访问诉求列表页或诉求详情页，When 页面渲染完成，Then 列表标题、详情说明、操作入口和辅助提示应以中文显示。
- Given 用户未登录，When 从首页进入公开主页、作品或诉求详情，Then 中文化不应新增登录阻断。

### FR-003 认证与工作台中文化

#### 需求说明

系统应将登录、注册、`studio` 工作台和 `inbox` 收件箱中的主要界面文案调整为中文，以便中文用户完成身份进入与内容管理。

#### 验收标准

- Given 用户访问登录页或注册页，When 页面渲染完成，Then 身份选择、表单标签、按钮和引导说明应以中文显示。
- Given 已登录用户进入 `studio`，When 浏览资料、作品和诉求管理页，Then 页面标题、操作入口、表单字段和辅助说明应以中文显示。
- Given 已登录用户进入 `inbox`，When 页面渲染完成，Then 收件箱标题、线程说明、空态或提示文案应以中文显示。

### FR-004 交互反馈与状态文案中文一致性

#### 需求说明

系统应将关键交互中的用户反馈统一为中文，避免出现按钮是中文但状态提示仍为英文、或同一概念在不同页面表达不一致的情况。

#### 验收标准

- Given 访客触发需要登录的点赞、收藏或联系动作，When 系统给出引导，Then 登录拦截与后续动作提示应以中文显示。
- Given 用户浏览任一空态页面或空态分区，When 当前无数据可展示，Then 空态标题或提示应以中文显示。
- Given 同一概念在多个页面出现，When 用户浏览首页、公开页与工作台，Then 关键术语应保持一致，不应一处中文一处英文混用。

### FR-005 页面 metadata 与品牌表达中文化

#### 需求说明

系统应将对外展示的页面标题、描述等 metadata 调整为适合中文用户理解的表达，同时允许品牌专名保留原始写法。

#### 验收标准

- Given 用户访问首页或关键公开页面，When 浏览器读取页面标题与描述，Then 主要对外描述应以中文表达。
- Given 页面中存在品牌名或不可避免的专名，When 与中文文案同时出现，Then 品牌名可保留原始写法，但不应导致页面整体以英文为主。

## 8. 非功能需求

### NFR-001 术语一致性

首页、公开页、认证页、工作台与收件箱中的关键术语应保持一致，例如“作品”“主页”“诉求”“联系”“收件箱”“登录”“注册”等不应在不同页面无规则漂移。

### NFR-002 现有结构稳定性

本轮中文化不应改变既有路由结构、页面层级和主要跳转关系；用户应仍可沿用现有浏览路径完成访问。

### NFR-003 浏览器可验证性

中文化结果不能只停留在代码层。完成实现后，必须通过浏览器实际访问至少以下三类路径中的代表页面进行验证：

- 首页
- 任一公开详情或公开主页
- 任一登录后页面

## 9. 外部接口与依赖

- 依赖现有 Next.js 页面与组件结构承载中文文案
- 依赖现有样本数据、站内交互状态和受保护路由继续工作
- 依赖浏览器验证环境访问本地运行中的应用

## 10. 约束

- 当前轮次默认只交付中文界面，不同时交付多语言切换能力
- 当前轮次应优先复用现有页面与组件结构，不以“中文化”为理由重做页面
- 当前轮次不修改现有 URL 结构
- 当前轮次允许保留品牌名 `Lens Archive` 等专名，但其他主要用户可见文案应尽量改为中文

## 11. 假设

- 假设当前站点主要面向中文用户，因此中文可作为默认界面语言
- 假设现有页面中的英文文案主要位于前端代码和样本数据中，可在当前轮次内集中调整
- 假设浏览器验证能够覆盖主要中文化结果，但不要求本轮建立自动化端到端测试体系

## 12. 开放问题

### 阻塞性开放问题

无

### 非阻塞开放问题

- 后续是否需要引入真正的国际化框架与语言切换能力，本轮不强制要求
- 后续是否需要同步把样本内容正文也进一步本地化到更强运营口径，本轮不强制要求
