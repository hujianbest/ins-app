# 首页与发现体验增强需求规格说明

- 状态: 已批准
- 主题: 首页与发现体验增强

## 1. 背景与问题陈述

首版已经交付了品牌化首页、公开主页、作品详情、诉求详情、认证入口、控制台、点赞收藏与站内联系闭环，但首页仍主要承担视觉首屏和静态入口作用，尚未形成更强的“发现”价值。

当前首页缺少更明确的内容编排策略、分区结构和内容兜底规则，导致：

- 访客进入首页后，能看到品牌感，但不容易继续深挖高质量内容
- 首页对作品、主页、诉求三类公开内容的承载仍偏轻
- 现有设计里提到的“人工精选优先 + 最新内容兜底”还没有被落实为明确需求

这一轮要解决的问题，是让首页从“视觉入口页”升级为“视觉入口 + 发现入口”，在不引入复杂个性化算法的前提下，提高公开内容的可发现性与继续浏览意愿。

## 2. 目标与成功标准

### 2.1 目标

- 强化首页作为公开内容发现入口的作用
- 让未登录访客也能快速看到值得点击的作品、主页和诉求
- 用清晰的编排规则降低首页内容为空、过旧或不均衡的风险
- 在不引入复杂推荐系统的前提下，交付可持续扩展的首页发现基线

### 2.2 成功标准

- 首页除 Hero 外，至少具备清晰的发现内容分区
- 访客可从首页直接进入精选作品、精选主页和精选诉求
- 当人工精选不足时，首页仍能用最新公开内容填充分区
- 首页的发现区对未登录访客完全开放，不依赖登录才能浏览

## 3. 用户角色与关键场景

### 3.1 主要用户

- 未登录访客：通过首页发现值得继续浏览的作品、主页和约拍诉求
- 已登录普通浏览者：在首页继续发现内容并进入公开详情
- 摄影师 / 模特创作者：其公开内容有机会被首页精选区曝光

### 3.2 关键场景

- 访客打开首页后，在 Hero 之下继续浏览“精选作品”“精选主页”“精选诉求”等发现模块
- 访客点击发现模块中的任一内容卡片，进入对应的作品详情、个人主页或诉求详情
- 当某一发现模块缺少人工精选内容时，系统使用最新公开内容兜底，避免首页空洞

## 4. 范围

- 首页新增或强化发现分区，至少覆盖：
  - 精选作品
  - 精选主页
  - 精选诉求
- 首页发现分区需要有明确的编排优先级
- 需要定义人工精选不足时的最新内容兜底规则
- 需要明确访客进入详情页的导航入口
- 需要保证首页发现增强不削弱现有全屏首屏视觉表达

## 5. 范围外内容

- 个性化推荐算法
- AI 推荐或复杂兴趣建模
- 推荐解释文案与推荐原因展示
- 独立运营后台或人工精选配置台
- A/B 实验平台
- 通知中心
- 搜索系统、筛选系统或标签体系的大规模扩展

## 6. 术语与定义

- 人工精选：由项目在代码内静态配置、种子数据或其他非独立运营后台方式预置的优先展示内容集合
- 最新内容兜底：当人工精选不足时，使用当前最新公开内容补足分区展示位
- 发现分区：Hero 下方用于承载公开内容发现的首页模块
- 最小展示数量：每个发现分区默认目标展示 3 条内容；若当前公开可用内容在去重后不足 3 条，则允许展示少于 3 条，且本轮不要求为不足数量制造空占位卡片

## 7. 功能需求

### FR-001 首页发现分区

#### 需求说明

系统应在首页 Hero 之下提供可继续浏览的发现分区，而不是只展示静态入口卡片。

发现分区至少包含：

- 精选作品区
- 精选主页区
- 精选诉求区

#### 验收标准

- Given 访客访问首页，When Hero 区域渲染完成后，Then 页面中应能看到至少三个独立的发现分区，分别承载作品、主页和诉求内容。
- Given 访客浏览首页发现分区，When 查看任一分区，Then 应能识别该分区展示的是哪类公开内容。

### FR-002 编排优先级与兜底规则

#### 需求说明

首页发现分区应采用“人工精选优先，最新公开内容兜底”的编排策略。

每个发现分区的目标最小展示数量为 3 条。

当某个分区内的人工精选内容不足时，系统应自动补充最新公开内容，直到达到该分区的最小展示数量，或在当前可用公开内容耗尽时停止补充。

最新内容兜底规则如下：

- 兜底内容按单一时间序倒序补充，当前轮次统一采用“公开内容的最新发布时间或最新可用更新时间”作为排序主键
- 已在当前分区人工精选中出现的内容，不得再次以兜底条目重复出现
- 同一条公开内容在同一分区中只能出现一次
- 当前轮次不要求限制“同一作者在同一分区出现多条内容”，只要其内容满足排序与去重规则即可
- 若去重后仍不足 3 条，则该分区允许少于 3 条展示，不要求补空卡

#### 验收标准

- Given 某发现分区已配置人工精选内容，When 首页渲染该分区，Then 人工精选内容应优先出现在分区中。
- Given 某发现分区的人工精选内容不足，When 首页渲染该分区，Then 系统应使用最新公开内容补足剩余展示位。
- Given 某发现分区没有任何人工精选内容，When 首页渲染该分区，Then 系统仍应使用最新公开内容输出该分区，而不是直接留空。
- Given 某条内容已经作为人工精选出现在当前分区，When 系统继续用最新内容兜底，Then 该内容不得在同一分区重复出现。
- Given 某分区去重后可用公开内容少于 3 条，When 首页渲染该分区，Then 该分区可展示少于 3 条真实内容，且本轮不要求显示空占位卡片。

### FR-003 首页到公开详情的发现入口

#### 需求说明

首页发现分区中的每条内容都必须提供明确的公开浏览入口。

不同内容类型的跳转目标如下：

- 作品卡片 -> 作品详情页
- 主页卡片 -> 摄影师主页或模特主页
- 诉求卡片 -> 诉求详情页

#### 验收标准

- Given 访客在首页看到作品卡片，When 点击卡片，Then 应进入对应作品详情页。
- Given 访客在首页看到主页卡片，When 点击卡片，Then 应进入对应公开主页。
- Given 访客在首页看到诉求卡片，When 点击卡片，Then 应进入对应诉求详情页。

### FR-004 公开可访问性

#### 需求说明

首页发现增强必须继续面向未登录访客开放，不得把发现浏览建立在登录前提上。

#### 验收标准

- Given 用户未登录，When 访问首页，Then 应能完整浏览发现分区和其中的公开入口。
- Given 用户未登录，When 从首页进入作品、主页或诉求详情，Then 不应因首页发现能力增强而新增登录阻断。

### FR-005 首屏视觉不退化

#### 需求说明

首页发现增强不应削弱现有首页的全屏视觉首屏表达。Hero 仍应保持首页的第一视觉焦点，发现分区位于其后的继续浏览层。

#### 验收标准

- Given 首页加载完成，When 用户进入页面首屏，Then 全屏 Hero 仍是最先呈现的核心视觉区域。
- Given 首页存在发现分区，When 用户查看首屏结构，Then 发现分区不应替代或挤占 Hero 的核心视觉角色。

## 8. 非功能需求

### NFR-001 公共可发现性

首页发现增强后的主要内容应继续通过公开页面可访问，不依赖登录、私有接口或受保护路由。

### NFR-002 内容新鲜度兜底

当人工精选内容不足时，首页仍应能通过最新公开内容补足，避免出现大面积空分区。

### NFR-003 视觉一致性

首页新增发现分区应与现有 Lens Archive 首页视觉基线保持一致，至少满足以下条件：

- 继续复用当前首页的深色高对比主视觉基调，不切换到独立主题
- 发现分区中的标题层级、正文层级与卡片式内容组织应延续现有首页的排版节奏
- 新增发现分区不应压缩 Hero 到失去“首屏主视觉”地位

## 9. 外部接口与依赖

- 依赖现有公开内容源：公开主页、公开作品、公开诉求
- 依赖首页内容编排数据源支持“人工精选 + 最新内容兜底”的读取能力

## 10. 约束

- 当前轮次不引入个性化推荐
- 当前轮次不要求独立运营后台
- 当前轮次允许通过代码内静态配置、种子数据或等价受控方式维护人工精选内容
- 当前轮次必须复用现有公开内容类型，而不是新增全新内容域
- 当前轮次不改变首页公共访问属性

## 11. 假设

- 假设当前项目可为首页提供至少少量人工精选数据或预置精选内容
- 假设当人工精选不足时，可从现有公开内容中识别“最新内容”作为兜底来源

## 12. 开放问题

### 阻塞性开放问题

无

### 非阻塞开放问题

- 后续是否需要加入“热门”“近期互动高”之类的规则排序维度，本轮不强制要求
- 后续是否需要加入更细的主题分组或风格分组，本轮不强制要求
- 后续是否需要限制同一作者在同一发现分区中的曝光条数，本轮不强制要求
