# 摄影作品展与约拍网站任务计划

- **状态：** 已批准
- **主题：** 摄影作品展与约拍平台
- **输入规格：** `docs/specs/2026-04-05-photo-showcase-platform-srs.md`
- **输入设计：** `docs/designs/2026-04-05-photo-showcase-platform-design.md`
- **关联评审：** `docs/reviews/tasks-review-photo-showcase-platform.md`
- **当前活跃任务：** 无

## 概述

本计划将首版交付拆为“基础设施与展示骨架 -> 公开展示链路 -> 账号与控制台 -> 互动闭环”四个里程碑，并优先保证：

- 尽快替换当前默认 Next.js 启动页
- 先建立可执行的测试基础设施，再进入持续实现
- 任何时刻只维护一个活跃任务

## 里程碑

### M1 基础设施与视觉骨架

- **目标：** 建立前端测试基础设施、替换默认启动页、形成首页与全局视觉基线。
- **包含任务：** `T1` `T2` `T3`
- **退出标准：** `web` 内具备可运行测试命令，默认首页已替换为品牌化首页骨架，基础元数据与视觉令牌已落地。

### M2 公开展示链路

- **目标：** 打通摄影师主页、模特主页、作品详情和约拍诉求的公开浏览链路。
- **包含任务：** `T4` `T5` `T6` `T7`
- **退出标准：** 访客可从首页进入公开主页、作品详情与诉求详情，页面均使用样本数据完成端到端导航。

### M3 账号与创作者控制台

- **目标：** 建立首版认证、角色进入与基础内容管理后台。
- **包含任务：** `T8` `T9` `T10`
- **退出标准：** 摄影师/模特可登录并进入控制台，完成主页编辑、作品管理和诉求管理的首版闭环。

### M4 互动与联系闭环

- **目标：** 完成点赞、收藏、站内联系与收件箱。
- **包含任务：** `T11` `T12`
- **退出标准：** 登录用户可从公开内容发起点赞、收藏和站内联系，被联系方可在站内看到会话。

## 任务拆解

### T1 建立 Web 端测试基础设施

- **目标：** 为 `web` 应用补齐 TypeScript/Next.js 前端测试基础设施，使后续任务可按 fail-first 纪律推进。
- **前置条件：** 已批准规格、设计和任务计划；当前 `web/package.json` 尚无测试脚本。
- **会触碰的工件：** `web/package.json`、测试配置文件、测试初始化文件、首个 smoke test 文件。
- **验证方式：** 能在 `web` 内运行统一测试命令，并看到至少一个前端 smoke test 被执行。
- **完成条件：** 测试命令可运行；最小 smoke test 可执行；不破坏现有 lint/build 入口。

### T2 替换默认首页为品牌化全屏首页骨架

- **目标：** 将默认 Next.js 启动页替换为符合规格的全屏 Hero + 精选入口首页骨架。
- **前置条件：** `T1` 完成。
- **会触碰的工件：** `web/src/app/page.tsx`、相关样式或组件文件。
- **验证方式：** 首页展示站点品牌、全屏视觉首屏、精选入口区域；首页 smoke test 或渲染测试通过。
- **完成条件：** 默认模板内容被移除；页面结构与规格中的首页体验一致；测试通过。

### T3 更新全局 metadata 与视觉基础

- **目标：** 更新站点标题、描述、基础字体与主题变量，形成统一视觉基线。
- **前置条件：** `T2` 完成。
- **会触碰的工件：** `web/src/app/layout.tsx`、`web/src/app/globals.css` 及相关样式文件。
- **验证方式：** 页面 metadata、全局样式变量和基础排版在本地可见且无构建错误。
- **完成条件：** 不再保留默认 “Create Next App” 元数据；站点具备统一视觉基线。

### T4 构建公开内容样本数据层

- **目标：** 提供首页、主页、作品和诉求页面可复用的本地样本数据与类型定义。
- **前置条件：** `T3` 完成。
- **会触碰的工件：** `web/src` 下共享数据与类型文件。
- **验证方式：** 公开页面可从共享样本数据读取内容；类型检查通过。
- **完成条件：** 后续公开展示页面不再依赖内联硬编码片段。

### T5 实现摄影师与模特公开主页

- **目标：** 建立 `/photographers/[slug]` 与 `/models/[slug]` 路由及沉浸式主页布局。
- **前置条件：** `T4` 完成。
- **会触碰的工件：** 路由页面、共享展示组件、样本数据。
- **验证方式：** 访客可访问两个角色主页并看到封面、简介、作品入口和联系入口。
- **完成条件：** 两类主页均可独立渲染，且结构符合设计文档。

### T6 实现作品详情页

- **目标：** 建立作品详情页，展示主图、作品说明、所属主页摘要与互动入口。
- **前置条件：** `T5` 完成。
- **会触碰的工件：** `/works/[workId]` 路由及其相关组件。
- **验证方式：** 可从主页进入作品详情；详情页显示作品主体与返回链路。
- **完成条件：** 作品详情公开可浏览，页面信息完整。

### T7 实现约拍诉求列表与详情页

- **目标：** 建立公开约拍诉求列表和详情页，支持从首页进入并查看城市、时间与发布者摘要。
- **前置条件：** `T4` 完成。
- **会触碰的工件：** `/opportunities`、`/opportunities/[postId]` 路由及组件。
- **验证方式：** 访客可浏览诉求列表、进入详情并看到站内联系入口。
- **完成条件：** 约拍诉求公开浏览链路闭合。

### T8 实现首版认证入口与角色注册流

- **目标：** 建立登录、注册和首版单主身份角色选择流程。
- **前置条件：** `T4` 完成。
- **会触碰的工件：** 认证页面、表单组件、会话与角色状态管理相关文件。
- **验证方式：** 摄影师/模特可完成角色注册并进入登录态。
- **完成条件：** 登录态可用于保护互动和控制台入口。

### T9 实现控制台中的主页与作品管理

- **目标：** 在 `/studio` 中提供主页编辑和作品管理首版界面。
- **前置条件：** `T8` 完成。
- **会触碰的工件：** 控制台路由、表单组件、状态管理与样本数据桥接代码。
- **验证方式：** 登录创作者可进入控制台、编辑主页资料并管理作品列表。
- **完成条件：** 创作者基础管理能力形成闭环。

### T10 实现诉求管理界面

- **目标：** 在控制台中提供约拍诉求的创建、编辑和下线首版界面。
- **前置条件：** `T8` 完成。
- **会触碰的工件：** 控制台诉求路由、表单组件与数据层。
- **验证方式：** 已登录创作者可创建至少包含城市与时间的诉求。
- **完成条件：** 诉求管理闭环可用。

### T11 实现点赞与主页收藏

- **目标：** 为公开作品和公开主页补齐登录后点赞/收藏行为。
- **前置条件：** `T6` 与 `T8` 完成。
- **会触碰的工件：** 互动按钮组件、登录拦截逻辑、状态管理层。
- **验证方式：** 未登录用户被引导登录；登录用户可成功执行点赞与收藏。
- **完成条件：** 互动状态可正确展示与切换。

### T12 实现站内联系与收件箱

- **目标：** 完成主页/作品/诉求页发起联系，以及 `/inbox` 收件箱查看联系线程。
- **前置条件：** `T6` `T7` `T8` 完成。
- **会触碰的工件：** 联系表单、线程列表、消息视图和来源对象上下文展示。
- **验证方式：** 登录用户可发起站内联系，被联系方可在收件箱看到线程。
- **完成条件：** 首版“展示 -> 联系”闭环完成。

## 完成定义

- 每个任务都有明确可运行的验证方式。
- 未通过验证的任务不得标记完成。
- 当前活跃任务完成并写回 fresh evidence 后，才能切换到下一个任务。
- 涉及页面改动的任务，至少需要对应渲染/交互验证证据。

## 验证策略

- 公共页面与共享组件优先使用前端单元/渲染测试。
- 核心导航与创作者主路径优先补充集成或端到端测试候选。
- 对当前仓库尚未存在的测试能力，先通过 `T1` 建立基础设施后再继续后续实现。
- 编码前需遵守 `web/AGENTS.md` 中关于 Next.js 版本文档的约束。

## 风险与顺序说明

- 当前 `web` 工程只有默认启动页，缺少可持续推进的测试能力，因此 `T1` 必须先做。
- `web/AGENTS.md` 提示当前 Next.js 版本存在与常见经验不同的破坏性变化，编码前需查阅本地 Next.js 文档。
- 互动与联系功能依赖登录态，因此应晚于公开展示链路和认证基础。

## 当前活跃任务选择规则

- 否则从依赖已满足的任务中选择编号最小者作为唯一活跃任务。
- 当前计划初始活跃任务为 `T1`。
